# Market River Real Data Design

## Goal

Use one real local narrative, `AI Infrastructure Demand`, to test two genuinely different market-history views.

## Data Mapping

- Narrative memory provides the durable river and its current interpretation.
- Morning/evening reports provide dated evidence events.
- Market scans provide confirmation, divergence, and unresolved market feedback.

## Views

- Version A is edited history. It groups evidence into a few regime-changing impact clusters and shows permanent changes in narrative water level.
- Version B is auditable history. It keeps individual evidence points visible and shows how supporting evidence, friction, and market divergence accumulate.

## Interaction

- Clicking an event opens a detail panel with the real date, source report type, evidence summary, affected companies, and interpretation.
- Version A exposes the evidence contained within each impact cluster.
- Version B allows filtering evidence by support, friction, and market response.

## Scope

This is a standalone HTML mockup only. It does not change the production Markets UI or server behavior.
